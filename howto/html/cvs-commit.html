3.6. 提交更新

一旦您完成了对一个文件的更改，并已使您的 XML 文件生效 (validated)，您就可以向 CVS 资源库提交您所做的更新。

在向资源库提交文件时，给您所做的更新作一个简短的注释是最基本的礼貌。您无需写明您所做的所有修改，这是 CVS 系统本身工作的一部分，但是您所做的注释有利于帮助工作人员在文档被更新后不查看源文档就能知道该文档被做了哪些修改。

请运行以下命令提交文件： 

$ cvs commit -m 'added explanation of the new ..blah.. feature' file1 [file2]

有时去掉 -m 参数会更方便一些，因为这样 CVS 将会打开一个文本编辑器来让你输入您的注释。当您希望添加更多的注释，并发觉无法用 -m 参数将这些注释写在一行里时，您便可以使用这种方式。

注意： 在提交文件前，请再次检查您的文件将被提交到您所工作的翻译模块下。phpdoc 模块仅包含英文文件，因此请不要将其它语言的文件提交到该模块下，除非得到了邮件列表上其他人的同意。在翻译模块的目录中，请不要放置只有英文内容的 XML 文件，这不便于翻译工作的维护，因此也请不要将任何纯英文的文件上传到翻译模块的目录树下。如果您不能确保您在提交文件的过程中不会发生错误，那么请到相关的邮件列表中垂询。
